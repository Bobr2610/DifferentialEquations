# Правила оформления академического учебника в Markdown

## 1. Структура документа

### Иерархия заголовков
- `#` - Заголовок документа (использовать только один раз)
- `##` - Основные разделы (главы, большие темы)
- `###` - Подразделы (определения, теоремы, примеры)
- `####` - Детальные подразделы (частные случаи, доказательства)
- `#####` - Мелкие подразделы (замечания, следствия)

### Разделители
- `---` - Разделитель между крупными разделами
- Использовать пустые строки для разделения абзацев

## 2. Типы контента и их оформление

### Определения
```
*Определение*: **Термин** - это определение термина.
```

### Теоремы и утверждения
```
*Теорема*: Название теоремы.
**Формулировка теоремы.**

*Доказательство*: Текст доказательства.
```

### Леммы и следствия
```
*Лемма*: Название леммы.
**Формулировка леммы.**

*Следствие*: Вывод из теоремы.
```

### Примеры
```
*Пример*: Описание примера.
```

### Замечания и утверждения
```
*Утверждение*: Важное утверждение.
```

### Замечания
```
*Замечание*: Дополнительная информация.
```

## 3. Форматирование текста

### Выделение
- **Жирный** - для терминов и названий (`**текст**`)
- *Курсив* - для переменных, обозначений (`*текст*`)
- `Моноширинный` - для кода, формул в тексте (`` `текст` ``)

### Списки
Нумерованные списки для последовательных шагов:
1. Первый шаг
2. Второй шаг

Маркированные списки для перечислений:
- Первый пункт
- Второй пункт

## 4. Математические выражения

### Встроенные формулы
Использовать `$...$` для формул в тексте: $f(x) = x^2$

### Отдельные формулы
Использовать `$$...$$` для выделенных формул:
$$ \int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi} $$

### Системы уравнений
```
$$
\begin{cases}
x + y = 1 \\
2x - y = 0
\end{cases}
$$
```

## 5. Структурирование контента

### Нумерация
- Теоремы, леммы, примеры: **Теорема 1.1:**, **Лемма 2.3:**
- Определения: **Определение 1.2:**
- Следствия: **Следствие 1.4:**

### Блоки контента
Для длинных определений использовать блоки:
```
> **Определение**: Длинное определение, которое занимает
> несколько строк и требует специального выделения.
```

### Таблицы для сравнений
```
| Свойство | Описание | Пример |
|----------|----------|--------|
| Свойство 1 | Описание | Пример |
```

## 6. Визуальные элементы

### Изображения
```
![Подпись к изображению](путь/к/изображению.png)
```

### Диаграммы
Использовать ASCII-art или внешние инструменты для диаграмм.

## 7. Соглашения по стилю

### Терминология
- **Жирный** для новых терминов при первом упоминании
- *Курсив* для обозначений множеств, функций
- `Моноширинный` для конкретных значений, констант

### Единообразие
- Все определения оформлять одинаково
- Все теоремы оформлять одинаково
- Все примеры оформлять одинаково

### Язык
- Использовать формальный академический язык
- Избегать сокращений в основном тексте
- Использовать точную математическую терминологию

## 8. Навигация

### Оглавление
Автоматически генерируемое оглавление в начале документа.

### Внутренние ссылки
```
[ссылка на раздел](#раздел)
```

### Сноски
```
Текст с сноской[^1]

[^1]: Текст сноски
```

## 9. Качество и читаемость

### Длина строк
- Не более 80-100 символов в строке
- Длинные формулы разбивать на несколько строк

### Пустое пространство
- Пустая строка перед каждым заголовком
- Пустая строка после каждого блока
- Пустая строка между абзацами

### Последовательность
- Соблюдать единую структуру во всем документе
- Использовать одинаковые обозначения
- Поддерживать логический порядок

## 10. Технические аспекты

### Кодировка
- UTF-8 для поддержки математических символов

### Совместимость
- Стандартный Markdown без специфических расширений
- MathJax для математических выражений

### Экспорт
- Возможность конвертации в PDF, HTML, LaTeX
